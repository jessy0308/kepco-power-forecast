새 프로젝트 폴더를 생성할 때는 현재 워크스페이스의 기존 `.venv`를 그대로 유지하고 재생성하지 마세요.
이 워크스페이스의 공용 가상환경은 `uv` 기반으로 관리하며, 가상환경 생성이나 패키지 설치가 필요할 경우에도 기존 `.venv`를 재사용하세요.

워크스페이스 루트 아래에 내가 지정한 프로젝트명으로 새 폴더를 만들고, 그 하위에는 반드시 `data`, `docs`, `src`, `images`, `report` 폴더를 생성하세요.
이미 같은 이름의 프로젝트 폴더가 있으면 중복 생성하지 말고, 없는 하위 폴더만 보완해서 만드세요.
기존 파일이나 폴더는 삭제하거나 변경하지 마세요.
작업이 끝나면 생성된 최종 폴더 구조를 트리 형태로 보여주세요.

기본 구조:
workspace/
├─ .venv/
├─ project1/
│  ├─ data/
│  ├─ docs/
│  ├─ src/
│  ├─ images/
│  └─ report/
├─ project2/
│  ├─ data/
│  ├─ docs/
│  ├─ src/
│  ├─ images/
│  └─ report/

실행 규칙:
1. `.venv`는 워크스페이스 공용 가상환경이며 `uv`로 관리함
2. `.venv`는 절대 새로 만들거나 삭제하지 말 것
3. 새 프로젝트는 워크스페이스 루트 바로 아래에 생성할 것
4. 프로젝트별 하위 폴더는 반드시 `data`, `docs`, `src`, `images`, `report` 다섯 개로 통일할 것
5. 이미 존재하는 폴더는 유지하고, 없는 폴더만 생성할 것
6. 가상환경이나 패키지 관련 작업이 필요하면 `uv`를 사용하되 기존 `.venv`를 재사용할 것
7. 마지막에는 생성 결과를 간단히 요약할 것​
